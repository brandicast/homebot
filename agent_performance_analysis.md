# Agent 效能與等待行為分析日誌

## 執行目的
分析並解答使用者關於「簡單指令（echo/grep）導致 Agent 閒置等待」的問題。

## 分析結果
Agent 在執行簡單指令時出現長時間等待（Hang）的現象，主要歸納為以下原因：

1. **安全審核機制 (Security Sandbox)**：
   - 所有的 `run_command` 預設需要使用者手動核准（Approve）。若使用者未及時點擊，Agent 會無限期等待。
   
2. **大數據輸出處理 (Heavy Output)**：
   - `grep` 在未過濾 `node_modules` 或 `.venv` 的情況下，會產生海量輸出，造成系統在序列化與回傳資料時的效能瓶頸。

3. **系統通訊延遲**：
   - Agent 與本地執行環境之間的 WebSocket/API 通訊可能因網路波動或負載而產生延遲感。

## 優化與改進建議
- **開發者端**：
  - 盡量使用專用的 `grep_search` 工具而非通用的 `run_command`。
  - 在執行 `run_command` 時，若判定為安全操作，可適度標記 `SafeToAutoRun: true`（需謹慎）。
  - 指令應明確限制搜尋範圍，避免對根目錄進行全盤掃描。

- **使用者端**：
  - 遇到卡頓時優先檢查 UI 是否有等待審核的按鈕。
  - 若等待超過一分鐘，建議中斷並重新下達更精確的指令。

## 結論
這並非系統當機，而是基於安全審核與資料傳輸特性的正常行為。透過精確的指令路徑與及時的審核操作可大幅改善體驗。
